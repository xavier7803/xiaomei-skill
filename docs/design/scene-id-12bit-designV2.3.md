# 小妹技能包内置语料标注与匹配机制 V2.3

## 一、设计概述
基于官方 12 位场景 ID 编码规范、核心维度优先级匹配、单个核心目的 P 标签关联、X 通配符模糊匹配设计，完全兼容现有语料规范和功能体系。
单条语料仅绑定 **1 个 12 位场景模糊码 + 1 个核心目的 P 标签**，依靠 X 通配实现多场景泛用；不强制次要维度填写 X，可自主选择精准编码或通配符；新增次要维度加权计分规则，在保证核心维度优先级的前提下，提升场景匹配贴合度。
本机制重点服务内置泛用语料的离线保底匹配，全程不涉及对话生成流程重构，与第一轮 LLM 结构化抽取机制 100% 无缝适配。

---

## 二、核心设计优势
✅ 性能极高：沿用纯字符串逐位匹配，O(1) 复杂度，无额外算法开销，响应延迟 <1ms
✅ 标注极简灵活：单条语料仅 1 个场景模糊码 + 1 个 P 标签，无多组白名单/黑名单，所有维度可自由选用精准码或 X
✅ 匹配精准分层：核心维度高权重优先，次要维度补充计分，兼顾防违和与场景贴合度
✅ 泛用性强：X 通配覆盖全场景需求，单条语料可适配海量场景
✅ 完全兼容：旧语料零改造兼容，未标注内容自动走通用兜底规则
✅ 无存储膨胀：单条语料仅新增少量标记，存储增量可忽略
✅ 离线鲁棒：完整兜底机制，杜绝空回复，完全脱离 LLM 亦可稳定运行

## 三、12 位场景 ID 编码规范
### 3.1 编码结构（6 个独立维度，12 位数字/X，从左到右）
| 位数   | 维度名称     | 编码范围    | 维度属性 | 标注规则                                   |
|--------|--------------|-------------|----------|--------------------------------------------|
| 1-2 位 | 时间段       | 00~99、X    | 核心维度 | 可填精准编码，或用 X 通配任意值             |
| 3-4 位 | 日期属性     | 00~99、X    | 核心维度 | 可填精准编码，或用 X 通配任意值             |
| 5-6 位 | 用户场景     | 00~99、X    | 次要维度 | 不强制通配，按语料适配范围自由填写          |
| 7-8 位 | 用户情绪     | 00~99、X    | 核心维度 | 可填精准编码，或用 X 通配任意值             |
| 9-10 位| 对话主题     | 00~99、X    | 次要维度 | 不强制通配，按语料适配范围自由填写          |
| 11-12 位| 特殊标识    | 00~99、X    | 弱维度   | 默认可用 X，节日/专属场景可填精准编码       |

## 四、语料库规范扩展
### 4.1 语料格式定义（唯一标准格式）
```
"语料内容 （等级要求; 12位场景模糊码; 单个P标签）"
```

### 4.2 规则说明
1. **唯一标注规则**：所有内置语料仅允许 1 个 12 位场景模糊码 + 1 个核心目的 P 标签，严禁多场景码、多 P 标签
2. **通配规则**：X = 该维度通配任意值；00 = 维度未知/无法判断，不代表通用
3. **维度标注自由**：核心/次要维度均可自由选用精准编码或 X，无强制约束
4. **缺省兼容规则**
   - 未标注场景码 → 默认 `XXXXXXXXXXXX`
   - 未标注 P 标签 → 默认 `P01-05-无目的闲聊`
5. **适配原则**：泛用语料多用 X，专属场景语料可提高精准编码比例

### 4.3 示例说明
```
# 泛用安慰语料
"抱抱你，别委屈啦，我一直都在～ （1; 0801XX09XXXX; P01-04）"

# 居家娱乐专属语料
"在家玩游戏也太快乐啦，一起唠唠呀～ （1; 0802010105XXXX; P01-02）"

# 全场景保底语料
"我一直在呀，随时都可以找我～ （1; XXXXXXXXXXXX; P01-05）"

## 五、匹配逻辑设计
### 5.1 整体匹配流程
```
1. 获取当前 12 位标准场景 ID + 核心目的 P 标签
   → 优先使用第一轮 LLM 输出；LLM 异常则本地规则生成
2. 筛选符合等级要求的语料
3. 目的过滤：仅保留 P 标签完全一致的语料（一票否决）
4. 场景匹配：校验当前场景 ID 与语料场景模糊码是否兼容
5. 全维度加权打分：核心维度 + 次要维度计分并排序
6. 空结果降级：无匹配语料 → 触发对应 P 标签全 X 保底语料
```

### 5.2 场景通配匹配算法
```python
def match_scene(current_scene_id: str, rule_scene_id: str) -> bool:
    """
    12位场景ID通配匹配（支持X）
    :param current_scene_id: LLM/本地生成的12位纯数字场景ID
    :param rule_scene_id: 语料绑定的12位场景模糊码
    :return: 是否匹配
    """
    if len(current_scene_id) != 12 or len(rule_scene_id) != 12:
        return False
    for i in range(12):
        rule_char = rule_scene_id[i].upper()
        if rule_char == 'X':
            continue
        if rule_char != current_scene_id[i]:
            return False
    return True
```

### 5.3 全维度加权打分算法
#### 计分规则
1. **核心维度（时间段、日期属性、用户情绪）**
   - 精准匹配：+10 分/位
   - X 通配：+5 分/位
2. **次要维度（用户场景、对话主题）**
   - 精准匹配：+5 分/位
   - X 通配：+2 分/位
3. **弱维度（特殊标识）**：仅参与匹配，不参与计分

#### 排序规则
- 按总得分从高到低选取
- 同分 → 核心维度精准匹配数多者优先
- 同分且核心维度一致 → 随机选取

```python
def calculate_score(current_scene_id: str, rule_scene_id: str) -> int:
    """
    全维度加权打分
    """
    score = 0
    # 字符串索引对应12位场景ID
    core_positions = [0, 1, 2, 3, 6, 7]      # 核心维度：时段、日期、情绪
    minor_positions = [4, 5, 8, 9]           # 次要维度：用户场景、对话主题

    # 核心维度计分
    for i in core_positions:
        r = rule_scene_id[i].upper()
        c = current_scene_id[i]
        if r == c:
            score += 10
        elif r == 'X':
            score += 5

    # 次要维度计分
    for i in minor_positions:
        r = rule_scene_id[i].upper()
        c = current_scene_id[i]
        if r == c:
            score += 5
        elif r == 'X':
            score += 2

    return score
```

## 六、场景 ID 生成逻辑
### 6.1 双模式生成机制
1. **默认模式（LLM 优先）**
   直接复用第一轮 LLM 输出的 **12 位纯数字场景 ID + 单选核心目的 P 标签**，无格式转换、无编码重映射
2. **兜底模式（本地离线）**
   LLM 调用失败/超时时，启用本地规则生成场景 ID，P 标签默认使用 `P01-05-无目的闲聊`

### 6.2 LLM 辅助生成（可选）
支持极简 Prompt 辅助生成，仅输出 1 组场景 ID + 1 个 P 标签，与本机制完全对齐

## 九、降级与容错机制
1. **一键开关**：可单独关闭场景匹配 / 目的匹配，快速回归通用回复
2. **格式容错**：语料标注异常时，自动忽略场景与 P 标签，不影响加载
3. **空匹配兜底**：无符合条件语料 → 触发对应 P 标签全 X 保底语料
4. **兼容模式**：未标注语料默认全场景通用 + P01-05

## 十、编码标准

3. 全系统编码、标签统一，无歧义、无冲突
4. 与第一轮 LLM 调用机制同源同构，100% 无缝适配
