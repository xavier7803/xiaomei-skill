# 场景库与匹配机制 开发指南（MVP版本）
## 一、开发目标
严格依据三个官方设计文档开发，实现完整场景库与匹配机制，作为MVP包含功能，100%兼容现有体系，无破坏性改动：
1. ✅ 实现12位场景ID本地生成逻辑（零外部依赖）
2. ✅ 实现通配符+白/黑名单双过滤匹配算法
3. ✅ 整合到现有对话流程，不影响原有功能
4. ✅ 配套场景管理命令，支持用户导入/导出/测试
5. ✅ 一键开关，可随时关闭回归原有逻辑

## 四、兼容保障方案
1. **现有语料完全兼容**：未标注场景标记的语料默认白名单为`XXXXXXXXXXXX`（全场景匹配），黑名单为空，完全不影响原有回复
2. **一键开关降级**：新增`scene_match_enabled`配置开关，出现问题可立即关闭功能，完全回归原有对话逻辑，无任何影响
3. **容错机制**：语料场景标记格式错误时自动忽略场景标记，按原有逻辑处理，不会导致语料加载失败或崩溃
4. **空结果自动降级**：场景匹配后无可用语料时，自动忽略场景限制，返回所有符合等级要求的语料，保证回复可用性

4. **性能测试**：1000条语料全量匹配耗时<2ms，不影响对话响应速度
