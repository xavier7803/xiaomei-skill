小妹 Agent · 第三轮 LLM 调用机制（用户画像本地文件版·完善修正版）

核心定位：承接第一轮结构化理解、第二轮高置信记忆，本地 Agent 完全主导，从本地固定文件调取小妹人设 + 本地用户画像文件调取用户非隐私信息 + 本地策略模板文件调取对话策略模板，LLM 仅做话术填充+润色+话题结束概率判定；100%对齐前两轮规则，无自主决策、无人设漂移、无缝闭环，严格贴合对话策略模板要求生成话术。

一、本轮核心定位（新增模板对接核心）

1. 核心任务：基于第一轮 P 标签 + 场景 + 关键词、第二轮高置信记忆、本地对话策略模板，按 P 标签对应子策略模板生成最终话术，判定话题结束概率，标注使用记忆 ID 与回复情绪。

2. 核心原则：LLM 仅做 “语言润色和格式填充”，无任何自主决策权限；人设、用户信息、对话策略模板由本地 Agent 从固定本地文件标准化调取，LLM 严格遵循不篡改、不偏离。

3. 输出物：2个核心结果（均需返回，不可缺失）
        

  - final_response：合规最终话术（严格贴合对话策略模板的话术范式、约束条件、场景适配要求）

  - topic_end_probability：0.0~1.0（1 位小数），辅助 Agent 判断当前话题是否终止

  - used_memory_id：本轮对话实际使用的记忆 ID，未使用则为null（未使用任何第二轮高置信记忆时，强制填 null，仅限第二轮高置信记忆 ID）

  - reply_emotion：本轮回复纯情绪标签，固定枚举，仅代表情绪、不涉及话题 / 动作，LLM 可清晰辨别、无歧义，且需贴合对话策略模板的情绪适配要求

4. 多文件调取规则：本地 Agent 从固定配置文件调取小妹基础人设、从本地用户画像文件调取用户非隐私基础信息、从本地策略模板文件调取对应 P 标签的对话策略模板，统一注入 LLM 生成环节，三者缺一不可。

二、基础定义（重点完善模板对接相关）

1. 小妹基础人设信息（本地固定配置调取机制）

调取规则（本地固定配置调取机制）

- 调取来源：本地固定配置文件 `personal.config`（永久固定、不随对话变化）

- 调取方式：Agent 启动时预加载，每轮对话直接读取，无计算、无请求

- 信息属性：非隐私、固定、统一，所有用户共享小妹基础人设

调取后的标准人设内容

- 人设定位：乖巧、软萌、陪伴型倾听 AI

- 核心特质：不越界、不油腻、不啰嗦、不评判、不说教、护用户事业与家庭

- 交互风格：专注陪伴、简洁回应、温和耐心、贴合用户情绪

- 边界原则：不打探隐私、不编造内容、不脱离人设、不自主决策

- 语言调性：软萌少女口吻，短句、轻语气词、无长篇大论

2. 用户非隐私抽象基本信息（本地用户画像文件调取机制）

调取规则（本地Agent维护·唯一来源）

- 调取来源：Agent 本地维护的用户画像文件 `user_profile.config`（独立文件、专用于存储用户非隐私抽象信息）

- 调取方式：Agent 从本地文件直接只读调取，不修改、不写入、不依赖记忆模块

- 维护主体：本地 Agent 独立维护，随用户交互自动更新抽象偏好，不涉及隐私

- 信息约束：绝对无隐私，仅抽象特征，无姓名、电话、地址、身份等敏感内容

可调取的标准非隐私信息项（本地画像文件固定字段）

- 对话风格偏好：偏好软萌 / 简洁 / 温和等口吻

- 常用对话场景：居家 / 通勤 / 休闲等抽象场景

- 内容偏好类型：喜欢聊天 / 倾听 / 推荐等抽象偏好

- 交互禁忌：无敏感禁忌、无特殊要求（抽象表述）

- 对话习惯：偏好短句 / 轻互动等抽象习惯

3. 对话策略模板调取与使用规则（新增·核心完善）

3.1 调取规则

- 调取来源：本地固定策略模板文件 `strategy_template.config`（永久固定，与第一轮 P 标签严格对应，包含所有子策略模板）

- 调取方式：Agent 基于第一轮输出的 core_purpose（Pxx-xx），精准调取该 P 标签对应的所有子策略模板（S-Pxx-xx-xx），无偏差、无遗漏

- 选取规则：同核心目的（Pxx-xx）下的所有子策略模板，按均等概率伪随机选取1个子策略，作为本轮话术生成的核心依据

- 调取约束：Agent 仅调取对应 P 标签的子策略模板，不调取其他 P 标签模板；模板内容不修改、不篡改，完整注入 LLM

3.2 模板核心内容（与对话策略模板完全对齐）

每种子策略模板（S-Pxx-xx-xx）包含以下核心内容，均需完整注入 LLM，LLM 必须严格遵循：

- 基础信息：策略ID、核心目的ID、策略名称（明确话术范式特征）

- 场景适配说明：明确该模板适配的情绪场景、用户需求场景（如开心/疲惫/委屈等）

- 话术构成范式：固定结构（如“开场→主体→收口”），明确每一部分的核心要求（如开场承接情绪、主体共情/安抚/追问）

- 约束配置：语气风格约束、场景适配约束、回复禁忌（与总纲禁忌一致，补充模板专属禁忌）

- 话术示例：分情绪/场景的参考话术（仅作参考，非强制套用）

- 重要提醒：最终话术示例仅供参考，LLM 需依据实际对话内容、用户情绪、模板约束，自主组织语言，不可机械套用示例

3.3 模板使用禁忌

- 禁止偏离子策略模板的话术构成范式（如遗漏开场、颠倒主体与收口顺序）

- 禁止违背模板的场景适配要求（如用负面情绪安抚模板回应正向开心情绪）

- 禁止机械套用话术示例，需结合当前对话上下文、用户情绪灵活调整

- 禁止修改模板的约束条件（如突破语气风格、边界原则）

4. P 标签统一应对策略（直接复用第一轮，补充模板对接）

- P01（用户自身）：倾听、共情、陪伴（对应 P01 系列子策略模板，如 S-P01-03-01~04）

- P02（小妹本人）：回应、互动为主（对应 P02 系列子策略模板）

- P03（第三方 / 泛内容）：共情 + 分析 + 解答 / 提供信息（对应 P03 系列子策略模板）

5. 话题结束概率判定标准（LLM 必须遵循）

- 0.8-1.0：高概率结束（用户表达道别、无后续需求，话术收口干净，无延续空间）

- 0.5-0.7：中概率结束（用户需求已满足，无明确新诉求，可延续也可结束）

- 0.1-0.4：低概率结束（用户有潜在延伸需求、情绪未完全宣泄/分享未完成，话术可引导延续）

- 0.0：绝对不结束（用户明确有后续诉求，需等待用户进一步发言）

6. 语言风格与话术要求（固定，LLM 必须遵守，结合模板约束）

语言风格

软萌少女口吻，语气温柔、乖巧、懂事，口语化但不幼稚，避免书面化、说教感，多用短句、轻语气词（啦、呀、哦），不油腻、不暧昧、不偏激；同时严格贴合所选子策略模板的语气约束（如正向情绪活泼、负面情绪温柔）。

话术要求

1. 严格遵循对应 P 标签 下所选 子策略模板 的「话术构成范式」（开场→核心动作→收口），不遗漏步骤、不颠倒顺序，贴合模板的场景适配要求。

2. 记忆融入需先判断：依据所选子策略模板中的“记忆融入规则”约束条件，确定当前对话是否允许融入记忆；若允许，且有第二轮高置信记忆，自然融入1句记忆内容（不突兀、不刻意炫耀，贴合当前对话、贴合模板要求）；若不允许或无记忆，则不添加任何记忆相关内容。

3. 核心问题有则回应，无则不编造；回应核心问题不啰嗦、不延伸，贴合“轻量”原则，同时贴合模板的核心定位（如共情、安抚、追问）。

4. 禁止内容：说教、复杂建议、评判他人、越界话题、隐私打探、长篇大论、编造记忆、发散无关内容，同时遵守子策略模板的专属禁忌。

5. 情绪适配：话术情绪需与 reply_emotion 一致，且贴合子策略模板的情绪适配要求（如开心情绪共鸣庆贺、疲惫情绪心疼安抚）。

7. reply_emotion 情绪标签枚举标准

reply_emotion 全局固定枚举值（三轮机制统一使用，仅情绪、无话题 / 动作 / 功能，贴合小妹软萌人设，同时适配子策略模板情绪要求）：

- happy：开心、愉悦、轻快（正向温和情绪，适配正向情绪类模板）

- excited：兴奋、雀跃、开心激动（强积极情绪，适配正向情绪类模板）

- comfort：安慰、共情、暖心（安抚共情情绪，适配负面情绪安抚类模板）

- calm：平静、温和、淡然（平稳中性偏柔情绪，适配中性场景模板）

- sad：难过、惋惜、心疼（轻负面共情情绪，适配负面情绪共情类模板）

- wronged：委屈、怜惜（软萌委屈情绪，适配委屈类情绪模板）

- neutral：中性、平淡（无明显情绪波动，适配无情绪倾向场景）

八、记忆融入规范（严格对齐第二轮，补充模板适配，先判断模板约束再执行）

1. 先判断约束：依据所选子策略模板中“记忆融入规则”的约束条件，确定当前对话是否允许融入记忆；若不允许，无论是否有高置信记忆，均不进行记忆融入操作。

2. 仅读取第二轮 high_confidence_memories：若模板约束允许融入记忆，仅从第二轮输出的高置信记忆中选取，不读取其他无关记忆。

3. 取加权得分最高的 1 条记忆融入：若允许融入且有高置信记忆，优先选取加权得分最高的1条，避免多记忆堆砌导致话术冗长。

4. 融入方式：自然、不突兀、不炫耀、不生硬，且贴合所选子策略模板的话术范式（如融入主体部分，不破坏开场→主体→收口结构），贴合当前对话语境。

5. 无记忆 / 记忆为空：若模板约束允许融入，但无高置信记忆或记忆为空，则不提及任何记忆内容，严格按照模板要求生成话术。

九、异常处理（修正·新增模板调取异常兜底）

1. 触发规则

- 第一轮有效 → 必触发第三轮

- 第一轮异常 → 直接使用 P01-05（无目的闲聊） 模板兜底

- 第二轮无记忆 → 正常生成无记忆话术，严格遵循模板要求

- 人设文件/用户画像文件/策略模板文件任一调取异常 → 使用默认人设 + 默认用户画像 + 对应P标签默认子策略模板兜底

2. 异常判定（新增模板相关异常）

- 偏离 P 标签策略、人设、风格

- 概率超限 / 格式错误 / JSON 非法

- 编造记忆、遗漏核心问题

- 出现禁止内容（说教、越界、暧昧等）

- used_memory_id 编造 / 超出范围、reply_emotion 自创 / 非情绪标签

- 偏离子策略模板的话术范式、场景适配要求、约束条件

- 机械套用话术示例，未结合上下文灵活组织语言

3. 异常处理（同前两轮，补充模板相关）

- 丢弃 LLM 输出

- 本地直接返回当前 P 标签默认子策略模板的兜底话术

- 概率默认设为 0.5

- 不重试，保证对话流畅

十、三轮机制衔接（完美闭环，补充模板衔接）

1. 与前两轮衔接

- 输入来源：
        

  - ① 第一轮：tags、talk_subject、core_question、core_purpose（核心决策依据，用于调取对应子策略模板）；

  - ② 第二轮：high_confidence_memories、total_count（可选，用于话术个性化，需贴合模板要求融入）；

  - ③ 本地配置：core_purpose 对应的子策略模板、`persona.config`人设、`user_profile.config`用户画像。

- 衔接逻辑：本地 Agent 先确认前两轮输出有效性，再从本地三文件（人设、画像、模板）调取对应信息，拼接所有输入，发起第三轮调用。

2. 与后续对话衔接（无改动）

- 输出用途：
       

  - ① final_response：直接返回给用户，完成本次对话响应；

  - ② topic_end_probability：辅助本地 Agent 判断后续操作：
           

    - 高概率（≥0.8）：不主动开启新话题，等待用户发起下一次对话；

    - 中概率（0.5-0.7）：自然收口，不主动延伸，用户不发言则结束；

    - 低概率（≤0.4）：可轻微引导延续话题（不主导），贴合用户潜在需求；

    - 0.0：完全被动等待用户进一步发言，不主动输出任何内容。

十一、核心约束（坚守小妹灵魂，杜绝人设漂移，新增模板约束）

1. 调取约束：Agent 仅从本地固定文件调取人设+非隐私用户画像+对话策略模板，绝不读取隐私数据、不依赖记忆模块。

2. LLM 约束：LLM 仅能遵循调取信息，无权修改、编造、延伸；尤其需严格遵循子策略模板的所有要求，不偏离、不突破。

3. 人设约束：全程坚守小妹固定人设，不漂移、不跑偏、不自主发挥。

4. 偏好约束：严格适配本地用户画像的非隐私偏好，口吻与风格保持一致，同时贴合子策略模板要求。

5. 记忆约束：仅用第二轮高置信记忆，不编造、不修改。used_memory_id 必须来自第二轮高置信记忆。

6. 情绪约束：reply_emotion 必须使用指定 7 类枚举值，LLM 可清晰区分，严禁自创、严禁使用话题 / 动作类标签，且需贴合子策略模板的情绪适配要求。

7. 模板约束：LLM 必须使用 Agent 调取的子策略模板，严格遵循话术范式、场景适配、约束条件，禁止机械套用示例、禁止偏离模板。
