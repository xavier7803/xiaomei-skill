# 小妹技能包 场景库功能设计方案
> ⚠️ 本设计原3位场景ID编码已废弃，现统一采用12位6独立维度编码标准，详细编码规则参考官方标准文档`scene-code-mapping.md`和`scene-id-12bit-design.md`
## 一、需求概述
为提升语料回复的场景适配性，解决不同场景下语料使用不当的问题，新增场景库功能：
1. 建立标准化场景库，每个场景分配唯一ID编码，包含对话时间、用户情绪、用户所在场景、对话主题等多维度特征
2. 扩展现有语料规范，每句语料附带允许使用的场景ID数组，仅在匹配场景下可被调用
3. 实现场景识别引擎：优先使用轻量规则识别，识别置信度不足时可调用LLM辅助识别
4. 对话流程新增场景匹配步骤，仅返回符合当前场景的语料，提升回复准确性

## 二、设计原则
完全遵循小妹技能包原有设计理念，保证最小改动、最大兼容：
1. **本地优先**：场景识别逻辑90%以上基于本地规则实现，不依赖LLM即可正常运行
2. **向后兼容**：完全兼容现有语料规范，未标注场景的语料默认全场景可用，不需要修改现有语料即可上线
3. **轻量可控**：场景识别逻辑简单高效，不引入过重的算法依赖，资源消耗可忽略
4. **降级保障**：场景识别失败或无匹配语料时，自动降级为原有回复逻辑，不影响基础对话体验
5. **可扩展性**：场景库和识别规则支持用户自定义扩展，无需修改核心代码

## 四、场景库设计
### 4.1 场景ID编码规则
统一采用官方标准12位6独立维度编码，所有维度完全独立，支持任意组合，详细编码映射参考`scene-code-mapping.md`：
| 位数 | 维度名称 | 编码范围 |
|------|----------|----------|
| 1-2位 | 时间段 | 00~99 |
| 3-4位 | 日期属性 | 00~99 |
| 5-6位 | 用户场景 | 00~99 |
| 7-8位 | 用户情绪 | 00~99 |
| 9-10位 | 对话主题 | 00~99 |
| 11-12位 | 特殊标识 | 00~99 |

匹配规则采用**通配符+白/黑名单双过滤**机制，详细匹配逻辑参考`scene-id-12bit-design.md`。

### 4.2 场景库存储格式
采用JSON格式存储，路径：`src/corpus/scene_definitions.json`，存储各维度编码映射，支持用户自定义扩展：
```json
{
  "time_period": {
    "00": "未知",
    "01": "凌晨(00-06点)",
    "02": "早上(06-12点)"
  },
  "date_type": {
    "00": "未知",
    "01": "工作日",
    "02": "周末"
  },
  "emotion": {
    "00": "未知",
    "01": "开心/兴奋",
    "02": "难过/低落"
  }
}
```

## 五、语料库扩展设计
### 5.1 语料规范扩展
完全兼容现有语料规范，在原有等级标记后新增场景双名单标记，整体格式：
```
"语料内容 （等级要求; 白名单场景码1[,白名单场景码2...]; 黑名单场景码1[,黑名单场景码2...]）"
```
- 通配符规则：场景码中的`X`表示该位任意数字，支持模糊匹配
- 兼容原有无场景标记的语料：默认白名单为`XXXXXXXXXXXX`（全匹配），黑名单为空，全场景可用
- 示例：
  ```json
  "早呀~ 今天工作日也要加油哦😘（1; 0201XX0001XX ; ）"
  ```
  表示该语料等级要求≥1级，仅匹配时间段=早上、日期属性=工作日、对话主题=问候的场景

### 5.2 解析逻辑
解析语料时自动提取场景信息：
1. 按`;`分割末尾括号内容，分别为等级要求、白名单列表、黑名单列表
2. 没有场景标记的默认全场景可用
3. 场景码自动校验有效性，无效码自动忽略
4. 黑名单优先级 > 白名单优先级，命中任意黑名单规则直接排除

## 六、场景识别引擎设计
### 6.1 识别流程
采用**规则优先，LLM辅助**的二级识别机制：
```
┌──────────────────────────────────────────────────────────┐
│                      输入用户消息                        │
└─────────────┬────────────────────────────────────────────┘
              │
┌─────────────▼────────────────────────────────────────────┐
│  规则识别：时间匹配 + 情绪匹配 + 关键词匹配              │
└─────────────┬────────────────────────────────────────────┘
              │
┌─────────────▼────────────────────────────────────────────┐
│  计算识别置信度：匹配维度越多，置信度越高                │
└─────────────┬────────────────────────────────────────────┘
              ├───────────置信度≥80%───────────┐
              │                                │
┌─────────────▼────────────┐       ┌─────────▼────────────────────────────────┐
│  返回规则识别的场景ID    │       │  LLM辅助识别（可选，可配置开关）           │
└──────────────────────────┘       └─────────┬────────────────────────────────┘
                                                 │
                                       ┌─────────▼────────────────────────────────┐
                                       │  LLM返回匹配的场景ID列表 + 置信度         │
                                       └─────────┬────────────────────────────────┘
                                                 │
                                       ┌─────────▼────────────────────────────────┐
                                       │  置信度≥60%返回LLM结果，否则返回通用场景   │
                                       └──────────────────────────────────────────┘
```
### 6.2 规则识别逻辑（本地实现，零依赖）
1. **时间匹配**：根据当前系统时间匹配场景的`time_range`，匹配加30分
2. **情绪匹配**：根据情绪识别结果匹配场景的`emotion_match`，匹配加30分
3. **关键词匹配**：用户消息包含场景的任意关键词，匹配加40分
4. **置信度计算**：三者得分相加，满分100分，≥80分认为识别成功

### 6.3 LLM辅助识别逻辑（可选，可配置关闭）
仅在规则识别置信度<80%时触发，调用LLM时使用极简Prompt，控制Token消耗：
```
你是场景识别助手，从以下场景列表中匹配用户当前对话最可能的3个场景，返回场景ID数组和置信度（0-100）。
场景列表：{所有场景的ID和名称JSON}
用户消息：{用户输入内容}
当前时间：{当前时间}
用户情绪：{识别到的情绪}
返回格式（仅返回JSON，不要其他内容）：{"scenes": ["020100000100", "020100000200"], "confidence": 85}
```
- LLM返回结果仅作为补充，置信度<60%时自动忽略，降级为全场景匹配
- 可通过配置开关完全关闭LLM辅助识别，纯规则模式即可正常运行

## 七、对话流程改造
### 7.1 语料筛选逻辑
1. 先按原有逻辑筛选符合等级要求的语料
2. 再根据识别到的场景ID列表，筛选出语料场景标记包含任意匹配场景ID的语料，或无场景标记的语料
3. 如果筛选结果为空，自动忽略场景限制，返回所有符合等级要求的语料，保证回复可用性

### 7.2 配置开关
新增两个配置项，可在技能管理页面调整：
| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| scene_match_enabled | boolean | true | 是否开启场景匹配功能，关闭后回归原有逻辑 |
| scene_llm_enabled | boolean | false | 是否开启LLM辅助场景识别，默认关闭纯规则运行 |

## 八、配套功能设计
### 8.1 命令扩展
新增3个场景库管理命令，和现有语料命令风格一致：
| 命令 | 功能 |
|------|------|
| /导出场景库 | 导出所有场景库配置，返回JSON格式 |
| /导入场景库 [append/cover] [JSON内容] | 导入场景库配置，支持追加/覆盖模式 |
| /重置场景库 | 重置场景库为默认状态 |

### 8.2 兼容现有导入导出功能
场景库作为特殊语料类型，自动纳入现有`/导出语料 all`、`/重置语料 all`命令的操作范围，无需额外开发。

## 十、风险评估和降级方案
### 10.1 风险点
- 场景识别准确率不足，导致语料匹配不当
- 新增逻辑引入Bug，影响现有对话功能
- 语料扩展后解析出错，导致语料加载失败

### 10.2 降级方案
1. **一键关闭开关**：出现问题可立即关闭场景匹配功能，回归原有稳定逻辑
2. **容错机制**：语料解析出错时自动忽略场景标记，按原有逻辑处理
3. **空结果降级**：场景筛选后无可用语料时，自动忽略场景限制，返回所有符合等级的语料
4. **单元测试覆盖**：所有新增逻辑100%覆盖单元测试，上线前充分验证兼容性
