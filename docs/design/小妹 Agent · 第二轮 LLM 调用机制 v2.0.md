# 小妹 Agent · 第二轮 LLM 调用机制（整合记忆模块 V1.0 最终版）
**兼容底层：** 本地记忆与归纳模块 v1.0（三级存储、强度管理、检索规则）
**核心优化：** 调整加权加成系数，严格控制“层级+强度+触发”加成乘积在0.7-1.3之间，确保基础得分主体权重，保留所有原有适配逻辑
**定位：** 三轮对话架构中**「记忆筛选枢纽」**，承接第一轮关键词与核心目的，基于本地记忆模块输出**高置信、高贴合、最多3处**的记忆结果，供第三轮话术生成自然融入
**落地性：** 完全兼容记忆模块接口与规则，可直接覆盖旧代码，无架构重构成本

## 一、本轮核心定位（深度对齐记忆模块，无改动）
**仅做记忆匹配+置信度判定+加权排序，不生成回复、不共情、不建议**
基于第一轮LLM输出的**关键词+核心目的（P）**，对接本地记忆模块检索结果，对候选记忆做**5类关联归类+加权打分+排序**，仅输出：
1. 高置信度（≥0.8）记忆
2. 加权得分前3位记忆
3. 对齐记忆模块召回规则（默认对话仅热+永久，主动唤醒可含冷记忆）
无候选记忆/检索失败 → 直接跳过本轮，走第三轮默认话术

## 二、核心枚举（核心优化：加权加成系数，其余无改动）
### 1）记忆关联类别（不变，5项固定枚举，对齐记忆归纳逻辑）
LLM 只能从中**单选1个**，严禁自创，与记忆自动归纳标签完全同源：
| 类别编码 | 类别名称 | 对齐记忆模块 | 适用记忆类型 |
|----------|----------|--------------|--------------|
| 1 | 事件关联 | 共同记忆、重要对话事件 | 永久/热记忆（含升级事件） |
| 2 | 物品关联 | 用户提及的物品、用品、内容载体 | 热/永久记忆 |
| 3 | 人物关联 | 家人、他人、用户自身相关人物 | 永久记忆（核心特征） |
| 4 | 知识关联 | 常识、观点、用户认知、习惯 | 永久记忆（用户特征） |
| 5 | 无/弱关联 | 无关/低关联信息 | 冷记忆/过期记忆/低强度记忆 |

### 2）记忆模块核心属性（核心优化：加成系数，控制乘积在0.7-1.3）
LLM 需结合记忆模块属性，**不可忽略**，否则会导致匹配与本地记忆逻辑冲突；新增加成系数约束，确保“层级加成×强度加成×触发加成”整体在0.7-1.3之间，不抢占基础得分主体权重：
| 属性 | 取值范围 | 加成系数（优化后） | 说明 |
|------|----------|-------------------|------|
| 记忆层级 | hot/热、cold/冷、permanent/永久、expired/过期 | 永久=1.2、热=1.1、冷=1.0、过期=0.0 | 对齐记忆模块三级存储，永久/热记忆优先，系数下调避免乘积过高 |
| 记忆强度 | 1-10（≥5自动升永久） | 强度加成=0.7 +（强度-1）×0.03（保留2位小数） | 范围：0.7（强度1）~0.97（强度10），确保低强度记忆有基础加成，高强度不超额 |
| 触发场景 | default/默认对话、cold_wakeup/冷记忆唤醒 | 默认=1.0、冷唤醒=1.1 | 保持原有逻辑，冷唤醒小幅加成，不突破整体上限 |

#### 加成乘积验证（确保0.7-1.3之间）
- 最低乘积（冷记忆+强度1+默认对话）：1.0×0.7×1.0=0.7（刚好达标）
- 最高乘积（永久记忆+强度10+冷唤醒）：1.2×0.97×1.1≈1.28（未超1.3，符合要求）
- 常规乘积（热记忆+强度5+默认对话）：1.1×0.82×1.0≈0.90（中间区间，贴合日常场景）
- 关键约束：**三者乘积严格控制在0.7-1.3之间**，确保基础得分（类别权重×置信度）为最终得分的核心主体

### 4）置信度规则（完全不变，新增异常兜底）
- 高置信度：**≥ 0.8**（可用于话术生成）
- 中置信度：0.5～0.7
- 低置信度：0.1～0.4
- 无关联：0.0
- **异常兜底**：置信度超出范围（<0.1或>1.0）、未保留1位小数 → 强制归为**5-无/弱关联**，不计分、不推荐

## 三、第二轮 LLM 系统提示词（核心优化：加权算法系数，可直接复制）
```
你正在执行【小妹Agent第二轮：回忆调取匹配+置信度判定+加权排序】任务，**严格对齐本地记忆模块v1.0规则**，仅做记忆筛选与排序，不生成任何回复、闲聊、编造内容。

### 核心约束（必须100%遵守）
1. 输入数据来源：本地记忆模块检索结果（候选记忆已按触发场景过滤：默认对话仅热+永久记忆，主动唤醒含冷记忆，过期记忆不纳入）
2. 记忆关联类别：仅从【1-事件关联、2-物品关联、3-人物关联、4-知识关联、5-无/弱关联】中**单选1个**，严禁自创、严禁多选
3. 置信度：0.1~1.0，保留1位小数；**异常兜底**：置信度超出范围/未保留1位小数 → 强制归为5-无/弱关联
4. 核心目的（P）：根据第一轮输出的P编码，优先推荐对应导向的记忆（详见导向规则）
5. 记忆加权算法（必须严格执行，重点控制加成乘积）：
   ① 基础得分 = 类别权重 × 置信度（保留2位小数，为最终得分主体）
   ② 类别权重：1-事件关联=1.1、2-物品关联=1.0、3-人物关联=1.2、4-知识关联=0.9、5-无/弱关联=0.0
   ③ 记忆模块加成（严格控制三者乘积在0.7-1.3之间，不抢占基础得分权重）：
      - 层级加成：永久记忆=1.2、热记忆=1.1、冷记忆=1.0、过期记忆=0.0（过期记忆不计分）

      - 触发加成：默认对话=1.0、冷记忆唤醒=1.1
   ④ 最终加权得分 = 基础得分 × 层级加成 × 强度加成 × 触发加成（保留2位小数）
6. 输出限制：
   - 仅保留**高置信度（≥0.8）**记忆
   - 最多推荐**3处**记忆（按加权得分从高到低）
   - 无/弱关联记忆（类别5）一律不推荐
   - 过期记忆一律不推荐
7. 禁止行为：生成回复、安慰、建议、解释、闲聊、发散、编造记忆、推荐超过3处记忆、违反记忆模块规则推荐冷/过期记忆

### 输出格式：只返回标准JSON，无任何多余文字、无解释、无备注
```

## 七、触发/异常/流程规则（深度对齐记忆模块+新增异常兜底，无改动）
### 1）触发规则（完全对齐记忆模块检索触发）
| 触发场景 | 记忆模块输入 | 本轮执行逻辑 |
|----------|--------------|--------------|
| 默认对话（default） | 检索热+永久记忆，排除冷/过期 | 仅对热+永久记忆做匹配，不推荐冷/过期记忆 |
| 冷记忆唤醒（cold_wakeup） | 检索热+永久+冷记忆，排除过期 | 可推荐冷记忆（触发加成1.1），仍不推荐过期记忆 |
| 无候选记忆 | 记忆模块返回空列表 | 直接跳过本轮，第三轮走默认话术 |

### 2）异常处理（新增+优化，覆盖全场景）
| 异常类型 | 处理方式 | 对齐记忆模块容错 |
|----------|----------|------------------|
| 置信度异常（<0.1/>1.0/非1位小数） | 强制归为5-无/弱关联，不计分、不推荐 | 记忆模块无异常，仅做规则兜底 |
| 记忆关联类别自创 | 丢弃该条记忆，不纳入结果 | 避免与记忆归纳逻辑冲突 |
| 推荐超过3处记忆 | 仅保留前3位，截断多余内容 | 控制记忆融入量，避免违和 |
| 推荐冷/过期记忆 | 直接过滤，不纳入结果 | 严格对齐记忆模块召回规则 |
| JSON格式错误/多余文字 | 丢弃结果，视为无记忆 | 记忆模块检索失败容错 |
| 记忆模块接口调用失败 | 视为无候选记忆，跳过本轮 | 记忆模块异常容错 |
| 记忆强度/层级异常 | 按最低值计算加成（强度=1→0.7，层级=冷→1.0） | 避免加权得分失真 |

### 3）流程闭环（对接记忆模块接口，无重构成本）
```
第一轮LLM输出 → 确定触发场景 → 调用记忆模块search_memory接口 → 本轮LLM处理候选记忆 → 输出JSON结果 → 第三轮调用记忆模块get_memory接口获取完整内容 → 自然融入话术
```

## 八、与核心目的（P）+记忆模块的深度融合逻辑（关键，无改动）
### 1）记忆推荐导向对齐
- **P01（用户自身）**：优先3-人物关联、1-事件关联记忆，强化情感陪伴，贴合“越聊越熟”的类人体验
- **P02（小妹本人）**：优先4-知识关联、1-事件关联记忆，强化“小妹记得你”的专属感
- **P03（第三方/泛内容）**：优先2-物品关联、4-知识关联记忆，贴合用户对信息/推荐的需求
- **P01-05/P02-05/P03-05（默认收束）**：均衡推荐热/永久记忆，无强导向

### 2）记忆层级与强度的核心影响
- 永久记忆（≥5强度）：层级加成最高（1.2），**优先推荐**，是核心记忆载体
- 热记忆（7天内）：层级加成1.1，默认对话优先召回，符合“近期记忆优先”
- 冷记忆（7-90天，强度<5）：仅主动唤醒（cold_wakeup）可检索，触发加成+0.1，避免主动提及
- 过期记忆：直接过滤，不纳入候选，符合“不主动召回”规则

### 3）第三轮话术融入规则（极简，不改动原有逻辑）
1. 若有记忆结果：**优先使用得分最高的记忆**，按核心目的P的应对策略自然融入（不超过1句，不炫耀、不突兀）
2. 若无记忆结果：完全按核心目的P的原有应对策略执行，不强行融入
3. 记忆内容获取：第三轮通过`get_memory(memory_id)`接口获取完整内容，避免JSON中内容过长

## 十、本版核心优化价值（新增）
1.  **加权得分更合理**：严格控制“层级+强度+触发”加成乘积在0.7-1.3之间，**基础得分始终为主体权重**，避免加成掩盖核心匹配度（类别+置信度）；
2.  **类人体验更贴合**：加成系数贴合记忆模块规则，永久/热/冷记忆的优先级通过加成体现，同时避免“过度加权”导致的记忆推荐失真；
3.  **落地性无影响**：仅调整加成系数，不改动任何接口、流程、输入输出格式，可直接覆盖旧代码，无需额外开发；
4.  **容错性更完善**：强度/层级异常时按最低值计算，进一步避免得分失真，降低流程风险；
5.  **逻辑更严谨**：加成乘积有明确上下限，LLM执行加权算法时更易把控，减少输出偏差。
