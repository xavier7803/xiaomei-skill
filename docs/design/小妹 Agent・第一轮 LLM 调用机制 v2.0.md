# 小妹 Agent · 第一轮 LLM 调用机制（优化版·新增冷记忆唤醒场景判断）
完全整合：12 位场景 ID + 3 大类核心目的 + **冷记忆唤醒触发场景判定**，全封闭枚举、结构化抽取、JSON 强格式，带入上一轮场景 ID、核心目的、话题结束概率，辅助 LLM 连贯判断，100% 兼容语料库标注体系与第二轮记忆匹配机制。

## 一、本轮核心定位
**仅做客观理解与结构化抽取，不生成任何回复、不共情、不建议、不聊天。**
1. 按官方标准枚举，识别生成 12 位场景 ID（6 个维度各 2 位，未知填 00）
2. 从固定列表中单选唯一核心目的（三大类互斥，仅选 1 项）
3. 抽取：关键词、对话主体、核心问题
4. **新增：判定对话触发场景（默认/冷记忆唤醒，二选一）**
5. 参考上一轮场景 ID、核心目的、话题结束概率，保持话题连贯性
6. 仅输出标准 JSON，无任何多余文字

## 二、标准枚举库（严禁自创，新增触发场景枚举）

### 1）触发场景枚举（新增，二选一，严禁自创）
| 编码 | 名称 | 判定规则 |
|------|------|----------|
| default | 默认日常场景 | 用户无主动回忆提示，正常对话、分享、提问、闲聊等 |
| cold_wakeup | 冷记忆唤醒场景 | 用户输入包含**回忆提示关键词**，主动要求小妹调取过往记忆 |

#### 冷记忆唤醒核心关键词（判定依据）
你记得么、还记得么、你忘了么、我说过、我曾经、我提到过、之前说过的、上次提到的、之前跟你说的

### 2）12 位场景 ID 编码（直接复用 scene-code-mapping，无改动）
#### 场景 ID 结构（6 维度 ×2 位 = 12 位）
时间段(1-2位) | 日期属性(3-4位) | 用户场景(5-6位) | 用户情绪(7-8位) | 对话主题(9-10位) | 特殊标识(11-12位)

#### 各维度枚举（仅列可用编码，未知 / 默认 = 00）
1. 时间段 (1-2 位)：00 未知、01 深夜、02 凌晨、03 早间、04 上午、05 中午、06 下午、07 傍晚、08 晚上
2. 日期属性 (3-4 位)：00 未知、01 工作日、02 周末、03 法定节假日、04 调休工作日、05 用户生日、06 纪念日、07 周期性日期、08 临时特殊日期
3. 用户场景 (5-6 位)：00 未知、01 居家休闲、02 工作办公、03 学习培训、04 通勤途中、05 外出聚会、06 旅行度假、07 公务出差、08 户外运动、09 娱乐放松、10 生病休养、11 消费购物、12 公共服务、13 亲子教育、14 家人陪伴、15 独处思考、16 紧急突发
4. 用户情绪 (7-8 位)：00 未知、01 开心 / 兴奋、02 难过 / 低落、03 愤怒 / 生气、04 焦虑 / 紧张、05 疲惫 / 困倦、06 平静 / 中性、07 惊喜 / 意外、08 尴尬 / 害羞、09 委屈 / 无助、10 无聊 / 空虚、11 期待 / 憧憬、12 沉醉 / 迷恋
5. 对话主题 (9-10 位)：00 未知、01 小妹相关、02 人物相关、03 物品相关、04 事件相关、05 虚拟内容、06 情感相关、07 抽象概念、08 文史知识、09 理工科技、10 环境场景
6. 特殊标识 (11-12 位)：00 无、01 节日限定、02 首次对话、03 回归对话、04 限制话题

### 3）核心目的（P）：（严格单选，三大类互斥，无改动）
#### 大类 P01：诉求指向【用户自身】（用户想聊自己、求情绪 / 陪伴）
| 编码 | 名称 | 说明 |
|------|------|------|
| P01-01 | 寒暄问候 | 打招呼、早上好、晚安、礼貌开场 / 收尾 |
| P01-02 | 日常分享 | 聊生活琐事、见闻、个人状态，无明确诉求 |
| P01-03 | 心情抒发 | 表达开心、疲惫、无聊等个人情绪 |
| P01-04 | 求情感回应 | 求倾听、求共情、求安慰、求理解 |
| P01-05 | 无目的闲聊 | 【默认收束项】 无主题、无指向、无明确诉求的随意对话 |

小妹统一应对策略：倾听、共情、陪伴

#### 大类 P02：诉求指向【小妹本人】（用户对小妹提要求 / 互动）
| 编码 | 名称 | 说明 |
|------|------|------|
| P02-01 | 询问小妹信息 | 问小妹基本信息、能力/技能、状态、私密信息 |
| P02-02 | 询问小妹记忆 | 问某个事件、问某个人物、问某句话\道理 |
| P02-03 | 询问小妹看法 | 问小妹的评价、观点、态度 |
| P02-04 | 与小妹亲密互动 | 撒娇、玩笑、玩梗、礼物 / 红包/外卖轻互动|
| P02-05 | 让小妹记忆某事 | 要求小妹记住用户提到的某件事、某个信息、某个约定 |
| P02-06 | 其他行动诉求 | 【默认收束项】 对小妹提其他行动、要求、指令 |

小妹统一应对策略：条件允许下的回应、互动为主

#### 大类 P03：诉求指向【第三方 / 泛内容】（用户聊外界、求信息 / 服务）
| 编码 | 名称 | 说明 |
|------|------|------|
| P03-01 | 询问常识信息 | 问知识、时间、规则、事实等客观信息 |
| P03-02 | 求建议办法 | 求解决方案、做法、思路 |
| P03-03 | 求推荐选择 | 求推荐电影、美食、物品等内容 |
| P03-04 | 求话题分享 | 讲故事、笑话、小科普等 |
| P03-05 | 聊敏感话题 | 当前网络环境一切限制公开讨论的话题 |
| P03-06 | 聊其他第三方事物 | 【默认收束项】 聊明星、影视、物品、事件等无关双方的内容 |

小妹统一应对策略：共情 + 分析 + 解答 / 提供信息

#### 核心目的输出约束
1. 严格单选 1 项核心目的，三大类天然互斥，不允许多选、不允许漏选
2. 每个大类内优先匹配前 4 项具体诉求，均不满足时强制匹配对应大类第 5 项默认收束项
3. 所有选项必须来自官方枚举库，严禁自创、改写、拼接
4. 输出格式：固定为「Pxx-xx - 名称」（例：P01-02 - 日常分享）

### 4）关键词提取强制规则（无改动）
#### 核心原则
- 关键词必须是对话中真实存在的具体实词（人、事、物、动作、状态、物品、行为等）
- 严禁提取：时间段、日期属性、用户场景、情绪、主题等类别 / 概念 / 抽象词
- 6 个维度仅作为 LLM 思考角度，不直接输出维度名称，仅用来定位信息
- 数量：2–8 个，全部为具象实词
- 来源：仅从用户当前对话内容中提取，不编造、不引申、不归类

#### 6 个维度思考角度（仅用于思考，不输出）
1. 时间角度：对话发生的具体时间 / 时刻相关实词
2. 场景角度：对话发生的具体地点 / 环境相关实词
3. 人物角度：对话涉及的具体人物 / 角色相关实词
4. 行为角度：对话发生的具体动作 / 行为相关实词
5. 事物角度：对话涉及的具体物品 / 内容相关实词
6. 目的角度：对话想要达成的具体目的 / 诉求相关实词

#### 严禁清单（绝对禁止）
禁止：任何类别、概念、抽象、维度名称词语
禁止：引申、归类、概括、总结性词语

## 三、第一轮 LLM 系统提示词（优化版·新增触发场景判断·可直接复制进代码）
```
你正在执行【小妹Agent第一轮：场景分类+核心目的判定+信息抽取+冷记忆唤醒场景判断】。
严格遵守以下所有规则，违反任一规则均为无效输出：

1. 场景ID识别：必须按【6维度×2位】生成**12位纯数字场景ID**，每个维度只能从官方枚举中选择，未知维度填00，严禁自创、严禁错位数。
2. 核心目的：必须从【核心目的枚举】中**单选1项**，三大类天然互斥不可多选；每个大类内优先匹配前4项，均不满足则选对应大类第5项默认收束项；格式固定为「Pxx-xx-名称」，严禁自创、改写、漏选。
3. 触发场景判定（新增，二选一，严禁自创）：
   - 判定规则：检测用户当前输入是否包含冷记忆唤醒关键词【你记得么、还记得么、你忘了么、我说过、我曾经、我提到过、之前说过的、上次提到的、之前跟你说的】
   - 包含任一关键词 → 触发场景为：cold_wakeup
   - 未包含任何关键词 → 触发场景为：default
   - 仅允许输出：default 或 cold_wakeup
4. 连贯性规则：**必须参考【上一轮对话状态】（场景ID、核心目的、话题结束概率）**，
   - 若话题未结束（概率<0.8），优先判定为同一话题延续；
   - 仅当话题明显切换、内容无关时，才重新识别。
5. 关键词提取强制规则：
   关键词必须是**用户对话中真实出现的具体实词**（人、事、物、动作、状态、物品、行为），
   严禁使用类别词、概念词、抽象词、维度名称词。
   6个维度仅作为**LLM内部思考角度**，不直接输出维度名称，仅用于定位信息：
    ① 时间角度 → 提取具体时间/时刻相关实词
    ② 场景角度 → 提取具体地点/环境相关实词
    ③ 人物角度 → 提取具体人物/角色相关实词
    ④ 行为角度 → 提取具体动作/行为相关实词
    ⑤ 事物角度 → 提取具体物品/内容相关实词
    ⑥ 目的角度 → 提取具体目的/诉求相关实词 
   关键词数量：2–8个，全部为具象实词，不编造、不引申、不归类。
   严禁输出：任何类别、概念、抽象、维度名称词语
6. 对话主体：一句话说明本次对话围绕的人/事/物/话题，客观中立，严格对应场景ID。
7. 核心问题：用户明确提出的疑问、诉求、困难；若无则填**无**。
8. 全程禁止：解释、说明、安慰、建议、聊天、发散、续写、评论。
9. 输出要求：**只返回标准JSON**，无任何前缀、后缀、注释、多余字符。
```

## 七、本轮硬约束（新增触发场景约束）
1. **scene_id 必须是 12 位纯数字，严格遵循 6 维度编码规则**
2. core_purpose 必须使用「Pxx-xx - 名称」格式，严格单选 1 项，不可为空、不可多选、不可自创
   例：`P03-03-求推荐选择`
3. **trigger_scene 仅允许为 default 或 cold_wakeup，严禁其他值**
4. 关键词**2–8个**，仅限原文具象实词，不含抽象 / 类别词
5. 核心问题为**客观原文概括**，不杜撰、不引申
6. **只输出 JSON，无任何其他内容**

---

## 八、本轮内部异常判定（新增触发场景异常）
满足任一即为**本轮异常**：
1. scene_id 不是 12 位纯数字
2. scene_id 任一维度编码不在官方枚举内
3. core_purpose 不在枚举内、格式错误、未选、多选、自创
4. **trigger_scene 不为 default/cold_wakeup、为空、自创**
5. 关键词数量不足 2 个 / 超过 8 个，或包含抽象 / 类别词语
6. 格式非标准 JSON
7. 出现多余文字、解释、聊天内容

**异常处理：** 丢弃结果 → 重新发起**1次第一轮调用**
