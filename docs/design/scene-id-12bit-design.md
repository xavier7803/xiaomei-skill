# 小妹技能包 12位场景ID+双名单匹配机制 正式设计方案
## 一、设计概述
基于用户提出的12位场景ID编码规范、通配符匹配、白名单+黑名单双重过滤机制设计，完全兼容现有语料规范和功能体系，实现成本低，匹配效率高，扩展性极强，可大幅提升回复的场景适配性和共情能力。
## 二、核心设计优势
✅ **性能极高**：纯字符串逐位匹配，O(1)复杂度，无额外算法开销，响应延迟增加<1ms
✅ **灵活性极强**：通配符支持任意维度模糊匹配，双名单机制可覆盖所有场景限定需求
✅ **扩展性极强**：6个完全独立维度各有充足编码空间，支持任意维度组合匹配，未来新增维度无需修改匹配逻辑
✅ **完全兼容**：不影响现有语料，未标注场景的语料默认全场景可用，零改造上线
✅ **数据量极小**：每条语料仅增加两个短字符串，存储量增加<5%，无膨胀问题
## 三、12位场景ID编码规范
### 3.1 编码结构（6个完全独立维度，12位数字，从左到右）
已按照官方统一编码标准调整，所有维度完全独立，支持任意组合，详细映射表参考`scene-code-mapping.md`：
| 位数 | 维度名称 | 编码范围 | 独立性说明 |
|------|----------|----------|------------|
| 1-2位 | 时间段 | 00~99 | 独立维度，仅表示一天内的时间区间（早/中/晚/深夜等） |
| 3-4位 | 日期属性 | 00~99 | 独立维度，仅表示日期类型（工作日/周末/节假日等） |
| 5-6位 | 用户场景 | 00~99 | 独立维度，仅表示用户当前所在物理/行为场景 |
| 7-8位 | 用户情绪 | 00~99 | 独立维度，仅表示用户当前情绪状态 |
| 9-10位 | 对话主题 | 00~99 | 独立维度，仅表示当前对话的核心主题 |
| 11-12位 | 特殊标识 | 00~99 | 独立维度，仅表示特殊场景标记 |
### 3.2 编码示例
| 场景ID | 说明 |
|--------|------|
| 0201XX0001XX | 时间段=早上、日期属性=工作日、对话主题=问候，其他维度任意 |
| XXXXXX02XXXX | 用户情绪=难过/低落，其他维度任意 |
| XXXXXX03XXXX | 用户情绪=愤怒/生气，其他维度任意 |
| XX03XX000101 | 日期属性=节假日、对话主题=问候、特殊标识=节日限定，其他维度任意 |
## 四、语料库规范扩展
### 4.1 语料格式定义
完全兼容现有语料规范，**所有官方内置语料必须注明唯一确定的12位场景码**，场景码严格依据`scene-code-mapping.md`中的编码规则生成，无限制或通用的维度默认填充为`00`，整体格式：
```
"语料内容 （等级要求; 12位场景码）"
```
### 4.2 规则说明
1. **强制标注规则**：所有官方内置语料必须标注唯一12位纯数字场景码，不得缺省，自定义用户语料可选标注
2. **通用维度规则**：无限制或通用的维度默认填充`00`，场景码为纯12位数字，不使用通配符，编码标准完全对齐`scene-code-mapping.md`
3. **缺省兼容规则**：未标注场景码的用户自定义语料默认全场景可用，零改造兼容原有语料体系
4. **扩展规则（可选）**：对于需要多场景匹配或黑名单限制的复杂场景，可兼容原有双名单格式，场景码支持X通配符，格式为`（等级要求; 白名单场景码1[,白名单场景码2...]; 黑名单场景码1[,黑名单场景码2...]）`
### 4.3 示例说明
#### 示例：工作日早上问候场景（标准内置语料格式）
```
"早呀~ 今天工作日也要加油哦，我会一直陪你的😘 （1; 020100000100）"
```
- 等级要求：≥1级
- 场景码：`020100000100` → 按位解析（严格对齐`scene-code-mapping.md`）：
  - 1-2位（时间段）：`02`=早上
  - 3-4位（日期属性）：`01`=工作日
  - 5-6位（用户场景）：`00`=通用/无限制
  - 7-8位（用户情绪）：`00`=通用/无限制
  - 9-10位（对话主题）：`01`=问候
  - 11-12位（特殊标识）：`00`=通用/无限制
- 效果：仅在工作日早上问候场景下可使用，其他场景不触发
## 五、匹配逻辑设计
### 5.1 整体匹配流程
```
┌──────────────────────────────────────────────────────────┐
│  1. 生成当前对话的12位标准场景ID（无通配符，全数字）   │
└─────────────┬────────────────────────────────────────────┘
              │
┌─────────────▼────────────────────────────────────────────┐
│  2. 筛选符合等级要求的语料（原有逻辑不变）              │
└─────────────┬────────────────────────────────────────────┘
              │
┌─────────────▼────────────────────────────────────────────┐
│  3. 白名单过滤：保留满足任意一个白名单场景码的语料      │
└─────────────┬────────────────────────────────────────────┘
              │
┌─────────────▼────────────────────────────────────────────┐
│  4. 黑名单过滤：排除满足任意一个黑名单场景码的语料      │
└─────────────┬────────────────────────────────────────────┘
              │
┌─────────────▼────────────────────────────────────────────┐
│  5. 空结果降级：筛选结果为空时忽略场景限制，返回所有符合等级的语料 │
└──────────────────────────────────────────────────────────┘
```
### 5.2 核心匹配算法（通配符匹配）
```python
def match_scene(current_scene_id: str, rule_scene_id: str) -> bool:
    """
    检查当前场景ID是否匹配规则场景ID（支持X通配符）
    :param current_scene_id: 当前对话的12位纯数字场景ID，无通配符
    :param rule_scene_id: 规则场景ID，可包含X通配符
    """
    if len(current_scene_id) != 12 or len(rule_scene_id) != 12:
        return False
    for i in range(12):
        rule_char = rule_scene_id[i].upper()
        if rule_char == 'X':
            continue  # 通配符直接跳过
        if rule_char != current_scene_id[i]:
            return False
    return True
```
### 5.3 性能评估
- 单条语料匹配耗时：<0.01ms
- 按1000条语料计算，全量匹配耗时<1ms，完全不影响响应速度
- 内存占用：1000条语料的场景标记仅占用约10KB内存，可忽略

## 九、风险评估和降级方案
### 9.1 风险点
- 语料解析出错导致语料加载失败
- 匹配逻辑Bug导致回复异常
- 场景ID生成不准确导致匹配不当
### 9.2 降级方案
1. **一键关闭开关**：新增配置项`scene_match_enabled`，出现问题可立即关闭场景匹配功能，回归原有稳定逻辑
2. **容错机制**：语料解析出错时自动忽略场景标记，按原有逻辑处理，不会导致语料加载失败
3. **空结果降级**：场景过滤后无可用语料时，自动忽略场景限制，返回所有符合等级的语料，保证回复可用性
4. **兼容模式**：未标注场景的语料默认全场景可用，原有语料无需任何修改即可兼容
