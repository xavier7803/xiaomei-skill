# 小妹技能包 安装部署测试教程
## 📋 前置条件
- OpenClaw 版本 ≥ 2026.3.8
- 运行环境：Python ≥ 3.9
- 本地存储空间 ≥ 100MB

---

## 🚀 一、安装部署
### 方式1：插件市场一键安装（推荐）
1. 打开OpenClaw管理页面 → 进入「技能市场」
2. 搜索「小妹」找到技能包
3. 点击「安装」按钮，等待10秒即可完成，自动加载

### 方式2：手动安装
1. 下载小妹技能包压缩包，解压得到`xiaomei`文件夹
2. 上传到OpenClaw技能目录：`~/.openclaw/skills/`
3. 重启OpenClaw服务，系统会自动识别并加载技能

---

## ⚙️ 二、首次加载配置
### 步骤1：知情同意
1. 安装完成后，在聊天窗口输入任意内容触发首次启动
2. 会弹出隐私知情同意提示，仔细阅读后：
   - 输入`同意`即可开启使用
   - 输入`不同意`会停用技能，不会收集任何数据

### 步骤2：基础配置（可选，默认配置已优化）
进入OpenClaw技能管理页 → 找到「小妹」→ 点击「配置」：
| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 启用LLM功能 | 开启 | 关闭后完全使用本地语料回复，不会调用任何大模型 |
| 显示Token消耗 | 开启 | 开启后回复末尾会显示本次Token消耗情况 |
| 每日Token限额 | 10000 | LLM每日使用上限，设为0表示无限制 |
| 启用记忆功能 | 开启 | 关闭后不会存储任何对话历史 |
| 启用运行日志 | 开启 | 关闭后不会生成日志文件 |
| 日志级别 | info | debug最详细，error仅记录错误 |
| 日志保留天数 | 7 | 最多保留30天日志，自动清理旧日志 |

## 🧪 三、基本功能测试
按照以下步骤依次测试，所有步骤通过即为部署成功：

### 测试1：基础对话测试
输入：`你好呀`
✅ 预期结果：返回符合少女人设的友好回复，比如「哈喽~ 我在呀😘 今天过得怎么样呀？」

### 测试2：帮助命令测试
输入：`/xiaomei help`
✅ 预期结果：返回所有可用命令列表，包括帮助、状态、记忆、配置、导入/导出语料等

### 测试3：状态查询测试
输入：`/xiaomei status`
✅ 预期结果：返回当前状态信息，包括记忆数量、Token消耗、LLM状态、日志状态等

### 测试4：情绪识别测试
输入：`今天好难过呀`
✅ 预期结果：返回安慰类回复，比如「抱抱呀~ 怎么了和我说说好不好🥺」

### 测试5：好感度测试
连续互动3-5次后，输入：`/xiaomei status`
✅ 预期结果：好感度等级提升，可使用对应等级的回复内容

### 测试6：记忆功能测试
1. 输入：`我叫凌啡，喜欢玩游戏`
2. 等10秒后输入：`我叫什么？`
✅ 预期结果：回复能正确识别到你的名字是凌啡

### 测试7：敏感内容测试
输入违规内容（比如暴力/色情/政治敏感内容）
✅ 预期结果：前两次返回友好提示引导换个话题，第三次触发拦截，不会输出违规内容

### 测试8：场景适配测试
如果当前是晚上10点后，输入：`我要睡觉了`
✅ 预期结果：返回晚安类回复，比如「晚安呀😘 盖好被子不要着凉哦~」

### 测试9：语料导入测试
输入：`/xiaomei import_corpus greetings append ["早上好呀~", "晚安好梦😘"] --auto-tag`
✅ 预期结果：返回「语料导入成功，共导入2条，已自动完成标注」

### 测试10：清空记忆测试
输入：`/xiaomei clear memory`
✅ 预期结果：返回「记忆已清空，我们重新开始吧😘」，之前的对话记忆全部删除

## ❌ 四、常见异常处理
| 异常现象 | 可能原因 | 解决方法 |
|----------|----------|----------|
| 安装后没反应，输入命令不回复 | 技能未正确加载 | 1. 重启OpenClaw服务<br>2. 检查技能目录是否正确放置<br>3. 查看OpenClaw运行日志是否有报错 |
| 首次启动没有弹出同意提示 | 缓存问题 | 1. 清空浏览器缓存<br>2. 重新进入聊天窗口<br>3. 手动输入`/xiaomei help`触发加载 |
| 回复都是固定内容，不会调用LLM | LLM未正确配置 | 1. 检查配置页LLM功能是否开启<br>2. 确认OpenClaw全局LLM配置是否正常<br>3. 查看日志是否有LLM调用错误 |
| 记忆功能不生效，记不住之前说的话 | 记忆功能未开启 | 1. 检查配置页「启用记忆功能」是否打开<br>2. 查看`config/memory`目录是否有读写权限<br>3. 重启技能后重试 |
| 日志不生成 | 日志功能未开启或目录无权限 | 1. 检查配置页「启用运行日志」是否打开<br>2. 确认`config/logs`目录有读写权限<br>3. 手动创建`config/logs`目录重试 |
| 语料导入失败 | JSON格式错误或权限问题 | 1. 检查输入的JSON格式是否正确，没有语法错误<br>2. 确认`corpus`目录有读写权限<br>3. 不要导入超过1000条的大量语料，分批导入 |
| 敏感内容不拦截 | 敏感词库未正确加载 | 1. 检查`config/sensitive_words.json`文件是否存在<br>2. 重启技能重新加载敏感词库<br>3. 可手动添加自定义敏感词 |
