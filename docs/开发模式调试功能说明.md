# 开发模式调试功能说明
功能说明：对接OpenClaw内置调试面板，开发/测试阶段输出调试信息，方便排查问题
当前MVP测试阶段默认开启，正式上线可调整为默认关闭
---
## 开关控制
通过环境变量`XIAOMEI_DEVELOP_MODE`控制：
- 值为`true`（默认）：开启调试模式，所有调试信息输出到OpenClaw内置调试面板
- 值为`false`：关闭调试模式，不输出任何调试信息，不影响主流程性能
---
## 输出的调试信息内容
所有调试信息仅在本地OpenClaw面板展示，不会上传到任何外部服务器，完全符合隐私安全要求：
1. 对话流程信息：用户输入内容、最终回复内容、各模块处理过程
2. 情绪识别结果：识别到的情绪类型、置信度、是否判定为敏感内容
3. 私密度检测结果：话题等级、用户当前等级、是否有权限交流
4. 好感度变动：每次互动的加减分情况、当前好感度数值、当前等级
5. LLM调用情况：是否触发润色、token消耗、调用是否成功
6. 异常错误信息：各模块运行报错详情，方便定位Bug
---
## 使用方法
1. 打开OpenClaw的调试面板（默认在OpenClaw管理后台→技能管理→调试面板）
2. 启动小妹技能，和小妹聊天，所有调试信息会实时输出到面板
3. 测试阶段结束后，设置环境变量`XIAOMEI_DEVELOP_MODE=false`即可完全关闭调试输出，不影响用户体验
---
## 注意事项
- 调试模式仅在开发/测试阶段使用，正式发布给用户时建议关闭，避免不必要的信息输出影响性能
- 调试信息仅展示在本地OpenClaw面板，不会记录到日志文件，不会泄露用户隐私
- 如果没有OpenClaw调试环境，调试功能会自动失效，不会影响主功能运行
